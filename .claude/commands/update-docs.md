# Update Documentation

Edit or create documentation pages for the cssvg-icon site.

## Usage

```
/update-docs [slug]
```

Examples:
- `/update-docs` — list all docs and ask which to edit
- `/update-docs getting-started` — open that doc for editing

---

## Doc Pages

All docs live in `content/docs/` as Markdown files.
The nav order is defined in `lib/navigation-data.ts`.

| Slug | File | URL |
|---|---|---|
| `introduction` | `content/docs/introduction.md` | `/docs/introduction` |
| `getting-started` | `content/docs/getting-started.md` | `/docs/getting-started` |
| `usage` | `content/docs/usage.md` | `/docs/usage` |
| `architecture` | `content/docs/architecture.md` | `/docs/architecture` |
| `adding-icons` | `content/docs/adding-icons.md` | `/docs/adding-icons` |
| `design-system` | `content/docs/design-system.md` | `/docs/design-system` |

---

## Editing an Existing Doc

1. Read the current file: `content/docs/<slug>.md`
2. Make the requested changes
3. Save — the page re-renders automatically on next request
4. No other files need updating unless the page title changes

---

## Adding a New Doc Page

1. Create `content/docs/<new-slug>.md` with the content
2. Add an entry to `lib/navigation-data.ts` in the correct position:

```ts
export const DOC_NAV = [
  { name: "Introduction", href: "/docs/introduction" },
  // ... existing entries ...
  { name: "New Page Title", href: "/docs/<new-slug>" },  // ← add here
];
```

3. The sitemap auto-includes it — no changes to `app/sitemap.ts` needed
4. The doc page is auto-generated by `app/docs/[slug]/page.tsx` — no changes needed

---

## Markdown Guidelines

- Use `##` for main sections (these appear in the Table of Contents)
- Use `###` for subsections
- Code blocks with language tag: ` ```tsx `, ` ```bash `, ` ```json `
- Keep tone: concise, developer-focused, no fluff
- Site URL: `https://icon.cssvg.com`
- Editor URL: `https://cssvg.com`
- npm package: `cssvg-icons`

---

## Metadata

Metadata for doc pages is auto-generated in `app/docs/[slug]/page.tsx`:
- Title: `<Slug Title> | cssvg-icon Docs`
- Description: auto-generated from slug
- OG type: `article`
- Structured data: `BreadcrumbList` JSON-LD

To customise metadata for a specific doc, edit `generateMetadata()` in `app/docs/[slug]/page.tsx`.
